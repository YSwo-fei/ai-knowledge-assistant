# 数据库学习需求文档

## 1. 文档目的

本文件用于明确后续数据库学习的目标、范围、阶段安排和验收标准。

当前阶段先以 Python、FastAPI、文件读写、接口理解为主。  
等基础能力稳定后，再系统进入数据库专题学习。

这份文档相当于未来的“数据库学习路线图”。

---

## 2. 学习背景

当前项目方向是：

- AI 全栈
- AI 应用落地
- RAG 知识库问答系统

在这个方向里，数据库不是附属技能，而是核心基础设施之一。

数据库将承担的任务包括：

- 存储文档信息
- 存储聊天记录
- 存储会话信息
- 存储用户信息（后续扩展）
- 支撑后端接口的数据读写

因此，数据库学习的目标不是“背语法”，而是：

**具备为 AI 项目设计、实现和维护数据层的能力。**

---

## 3. 总体学习目标

数据库学习完成后，应达到以下目标：

1. 理解数据库的基本概念和核心作用
2. 能独立完成简单数据表设计
3. 能写常见 SQL 语句完成增删改查
4. 能使用 SQLAlchemy 在 Python 项目中操作数据库
5. 能在 FastAPI 项目中实现完整的数据持久化流程
6. 能为 RAG 项目设计基础数据结构
7. 能在面试中清晰表达数据库设计思路

---

## 5. 分阶段学习目标

## 第一阶段：数据库入门

### 目标

建立数据库基本认知，理解数据库为什么存在，以及最基础的操作方式。

### 需要掌握

- 表、行、列的含义
- 主键的作用
- 数据类型
- SQL 的基本概念
- 增删改查四类操作
- SQLite 的基本使用

### 阶段成果

完成后应具备以下能力：

- 能看懂简单的数据表结构
- 能解释主键、字段、记录分别是什么
- 能写基础 SQL 语句
- 能说清楚数据库和普通文件存储的区别

---

## 第二阶段：数据库接入项目

### 目标

把数据库真正接入当前 Python / FastAPI 项目中。

### 需要掌握

- 数据库连接配置
- SQLAlchemy ORM 基本使用
- 模型定义
- 会话管理
- 数据写入与查询
- FastAPI 中的数据库操作流程

### 阶段成果

完成后应具备以下能力：

- 能定义一张文档表
- 能实现文档创建接口
- 能实现文档列表接口
- 能实现文档详情接口
- 能解释 `commit()`、`refresh()`、`close()` 的作用

---

## 第三阶段：数据库强化

### 目标

从“会写”升级到“会设计、会分析、会扩展”。

### 需要掌握

- 更完整的 SQL 查询能力
- 表结构设计能力
- 字段命名能力
- 默认值和可空字段设计
- 数据关系设计

### 阶段成果

完成后应具备以下能力：

- 能根据业务需求独立设计简单表结构
- 能判断字段是否应该为空
- 能判断哪些字段应该设置默认值
- 能为项目设计多张表之间的关系

---

## 第四阶段：面向 AI 项目的数据库能力

### 目标

让数据库真正服务于 AI 应用和 RAG 项目。

### 需要掌握

- 文档信息存储
- 聊天消息存储
- 会话管理
- 用户信息管理
- 关系库与向量库的职责区分

### 阶段成果

完成后应具备以下能力：

- 能为知识库问答系统设计数据层
- 能区分“业务数据库”和“向量数据库”
- 能解释为什么聊天记录、文档、会话要拆成不同的表

---

## 6. 学习重点说明

本学习任务的重点不是：

- 死记硬背 SQL 全部语法
- 一开始就研究复杂数据库优化
- 一开始就研究分布式数据库

本学习任务的重点是：

- 先理解数据库思维
- 先学会项目中最常用的操作
- 先能把数据库稳定接入自己的后端系统

## 8. 验收标准

数据库专题学习完成后，应满足以下验收标准：

### 基础验收

- 能解释表、行、列、主键
- 能写基础 SQL 语句
- 能说明 SQLite 的用途

### 项目验收

- 能在 FastAPI 中写出连接数据库的代码
- 能完成至少一张表的完整 CRUD
- 能把接口数据写入数据库并读取返回

### 设计验收

- 能自己设计文档表结构
- 能自己设计聊天记录表结构
- 能说清楚字段设计原因

### 表达验收

- 能向面试官解释自己的数据库设计
- 能说明请求模型和数据库模型的区别
- 能说明 SQLite、MySQL、PostgreSQL 的大致区别

---

## 9. 当前建议

当前阶段暂不立刻展开完整数据库专题训练。

建议顺序：

1. 先把 Python 基础、FastAPI 基础、请求与响应、接口调试彻底搞稳
2. 再进入数据库专题学习
3. 数据库基础稳定后，再回到项目中完成真正的数据持久化

这样可以避免现在同时卡在：

- Python 语法
- Web 接口
- 数据库概念
- ORM 操作

多个层面叠在一起导致理解混乱。

---

## 10. 未来执行建议

后续进入数据库专题时，建议按以下顺序推进：

1. 数据库概念课
2. SQL 基础课
3. SQLite 实战课
4. SQLAlchemy ORM 实战课
5. FastAPI + SQLite 项目实战
6. 文档表、聊天表、会话表设计
7. 数据库强化与项目重构

---

## 11. 一句话总结

数据库学习的最终目标不是成为数据库专家，而是：

**成为一个能为 AI 项目独立设计并实现数据层的开发者。**
